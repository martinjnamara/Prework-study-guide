HTML

  • The DOCTYPE is case insensitive
  • The head element contains information about the webpage
  • The head element appears toward the top of the page
  • The title element represents the document's title or name
  • The header exists within the body element
  • The header and contains navigational and introductory content
  • The body element represents the visible content shown to the user
  • The image element allows image representation within the body element
  • The image within an image element is referenced by a src attribute
  • The src attribute defines the URL/location of the image to be used
  • The src attribute is mandatory for the image element (img)
  • The heading tags, H1 through H6 should be used in descending order
  • The main element represents the main content of the body of a document or application
  • The main content area holds content directly related to the central topic of a document
  • The section element represents a generic section of a document or application
  • A section is a thematic grouping of content
  • The footer element can hold info such as author, copyright, contact, sitemap etc..

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Classes allows CSS to select and access specific elements via the class selector
  • display, determines the type of box generated for an element
  • text-align describes how a block's contents are horizontally aligned
  • width specifies the width of the content area, padding area or border area
  • margin is a property to set values for the thickness of the margin area
  • border is a property for setting border width, style, and colour
  • box-shadow attaches one or more drop-shadows to the box
  • height specifies the height of the content area, padding area or border area
  • padding-left is a property to set values for the thickness of the padding area

Git

  • git init - initialise an existing directory as a Git repository
  • git clone [url] - retrieve an entire repository from a hosted location using a URL
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: All files are staged; git add file - add a file to the next commit
  • git push origin feature/html: git pushes local branch to repository
  • git pull origin main: fetches code from the main branch on the remote repository
  • gitcommit -m "commit message": staged changes are sent to repository with a message
  • git add -A: All files are staged
  • git reset [file]unstage a file while kepping the changes in current working directory
  • git diff --staged shows a diff of what is staged but not yet committed
  • git diff - shows a diff of what is changed but not staged
  • git log - show all commits in the current branches history

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • The console.log() method is used to print messages to the console
  • Most statements require a terminating semicolon
  • Variable must be declared before use
  • A function is defined with the keyword 'function name ();
  • Variables can be declared with var name = 10; let name = 10; const Pi =3.142
  • 'The length of a string is given by stringName.length'
  • 'Single line comments are preceded by //'
  • 'Multi line comments are enclosed within /* ---- */'
  • 'Math.random() - this methodof the function Math returns a floating point random number (< 1)'
  • 'Math.floor() method of function Math returns the largest integer <= given number'
