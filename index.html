HTML

  • The DOCTYPE is case insensitive
  • The head element contains information about the webpage
  • The head element appears toward the top of the page
  • The title element represents the document's title or name
  • The header exists within the body element
  • The header and contains navigational and introductory content
  • The body element represents the visible content shown to the user
  • The image element allows image representation within the body element
  • The image within an image element is referenced by a src attribute
  • The src attribute defines the URL/location of the image to be used
  • The src attribute is mandatory for the image element (img)
  • the heading tags, H1 through H6 should be used in descending order
  • The main element represents the main content of the body of a document or application
  • The main content area holds content directly related to the central topic of a document
  • The section element represents a generic section of a document or application
  • A section is a thematic grouping of content
  • The footer element can hold info such as author, copyright, contact, sitemap etc..

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git push origin feature/html: git pushes local branch to repository
  • git pull origin main: fetches code from main branch
  • gitcommit -m "commit message": staged changes are sent to repository
  • git add -A: All files are staged
